# Security Frameworks

## What are Security Frameworks?

Security frameworks are guidelines used to build plans that help mitigate risks and threats to data and privacy.

They provide a structured approach to implementing a **security lifecycle**, which is a constantly evolving set of policies and standards that define how an organization:

- Manages risks
- Follows established guidelines
- Meets regulatory compliance (laws and regulations)

Different security frameworks are used to manage different types of organizational and regulatory compliance risks.

---

# Purpose of Security Frameworks

The main purposes of security frameworks are:

- Protect Personally Identifiable Information (PII)
- Secure financial information
- Identify security weaknesses
- Manage organizational risks
- Align security with business goals

---

# Four Core Components of Security Frameworks

## 1. Identify and Document Security Goals

The first step is identifying and documenting security goals.

**Example:**
An organization may aim to comply with the **General Data Protection Regulation (GDPR)**, a European Union law that gives citizens greater control over their personal data.

A security analyst may identify and document areas where the organization is not compliant with GDPR.

---

## 2. Set Guidelines to Achieve Security Goals

Organizations establish policies and guidelines to meet their security objectives.

**Example:**
To comply with GDPR, an organization may create new policies for handling data requests from users.

---

## 3. Implement Strong Security Processes

Organizations develop procedures that ensure security policies are followed.

**Example:**
A security analyst at a social media company may design procedures that allow verified users to update or delete their personal information while remaining GDPR compliant.

---

## 4. Monitor and Communicate Results

Organizations continuously monitor security controls and report findings.

**Example:**
A security analyst may monitor the organization's internal network and report potential GDPR-related security issues to a manager or compliance officer.

---

# Importance of Security Frameworks

Security frameworks help analysts:

- Document security policies
- Implement security procedures
- Follow established security standards
- Collaborate with other security team members
- Support organizational security goals

Understanding security frameworks is essential because they directly affect the work security analysts perform and how they collaborate with others.

---

# Security Controls

**Security controls** are safeguards designed to reduce specific security risks.

**Example:**
A company may require all employees to complete privacy training to reduce the risk of data breaches.

A security analyst may use software to automatically assign and track which employees have completed the training.

---

# Why Security Frameworks and Controls Matter

Security frameworks and controls help organizations:

- Reduce security risks
- Protect sensitive information
- Ensure regulatory compliance
- Maintain a strong security posture
- Encourage every employee to contribute to security

Understanding how security frameworks and controls work enables security analysts to support an organization's security goals and protect the people it serves.
