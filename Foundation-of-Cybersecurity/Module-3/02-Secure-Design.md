# CIA Triad

The **CIA Triad** is a foundational security model that helps organizations assess risk when designing systems and creating security policies.

**CIA** stands for:

- **Confidentiality**
- **Integrity**
- **Availability**

---

# 1. Confidentiality

**Confidentiality** ensures that only **authorized users** can access specific assets or data.

To maintain confidentiality, organizations use:

- Access controls
- User authentication
- Authorization policies
- Least privilege principles

**Example:**
Only employees with the appropriate permissions can access confidential customer records.

---

# 2. Integrity

**Integrity** ensures that data is:

- Correct
- Authentic
- Reliable
- Free from unauthorized modification

Security professionals use protections such as **encryption** and other security controls to help prevent data from being altered or tampered with.

**Example:**
A financial transaction should not be modified by an unauthorized user.

---

# 3. Availability

**Availability** ensures that authorized users can access data and systems whenever they need them.

Organizations maintain availability by using:

- Reliable infrastructure
- Backups
- Redundant systems
- Disaster recovery plans

**Example:**
Employees should be able to access business applications whenever required to perform their jobs.

---

# Assets

An **asset** is any item that has value to an organization.

The value of an asset is determined by the cost or impact associated with it.

**Example:**
An application that stores sensitive information such as:

- Social Security numbers
- Bank account information
- Customer records

is considered a valuable organizational asset.

---

# NIST Cybersecurity Framework

The **NIST Cybersecurity Framework (CSF)** is a voluntary framework consisting of:

- Standards
- Guidelines
- Best practices

Its purpose is to help organizations:

- Manage cybersecurity risks
- Reduce security threats
- Protect valuable assets
- Improve their overall security posture

Security teams commonly use the NIST CSF as a baseline for managing both short-term and long-term cybersecurity risks.

---

# Threat Actors

A **threat actor** is any individual or group that attempts to compromise an organization's security.

One of the most dangerous threat actors is a:

- **Disgruntled employee**

Disgruntled employees are particularly dangerous because they often:

- Already have access to sensitive information
- Know where valuable data is stored
- Understand internal systems and processes

---

# Reducing Insider Threat Risks

Organizations reduce insider threats by:

- Applying the **Availability** principle appropriately
- Following security frameworks and organizational policies
- Restricting employees to only the data they need to perform their jobs
- Implementing proper access controls

This follows the **Principle of Least Privilege**, ensuring users receive only the minimum level of access required.

---

# Importance of a Diverse Security Team

Threat actors operate from around the world and use many different attack techniques.

A **diverse cybersecurity workforce** helps organizations:

- Better understand attacker motivations
- Identify different attack patterns
- Improve risk assessment
- Develop stronger security strategies
- Reduce the impact of malicious activities

---

# Key Takeaways

- The **CIA Triad** consists of **Confidentiality, Integrity, and Availability**.
- **Confidentiality** protects data from unauthorized access.
- **Integrity** ensures data remains accurate and trustworthy.
- **Availability** ensures authorized users can access systems and data when needed.
- An **asset** is anything valuable to an organization.
- The **NIST Cybersecurity Framework** provides standards, guidelines, and best practices for managing cybersecurity risks.
- **Disgruntled employees** are significant insider threats because of their existing access and knowledge.
- Security frameworks and proper access controls help reduce insider risks and protect organizational assets.
