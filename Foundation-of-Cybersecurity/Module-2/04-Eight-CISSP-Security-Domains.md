# Eight CISSP Security Domains

The Certified Information Systems Security Professional (CISSP) framework organizes cybersecurity responsibilities into eight security domains. These domains provide a structured approach to understanding and managing information security within an organization.

---

## 1. Security and Risk Management

Security and Risk Management focuses on defining security goals and objectives, managing risks, ensuring compliance, maintaining business continuity, and adhering to legal and regulatory requirements.

### Key Areas

* Risk management
* Security policies and procedures
* Compliance and regulations
* Business continuity planning
* Legal and ethical responsibilities

### Example

Updating organizational policies to comply with regulations such as HIPAA.

---

## 2. Asset Security

Asset Security focuses on protecting physical and digital assets throughout their lifecycle, including storage, maintenance, retention, and secure disposal.

### Key Areas

* Data classification
* Asset management
* Data retention
* Secure storage
* Data destruction

### Example

Ensuring old devices are securely wiped and destroyed before disposal.

---

## 3. Security Architecture and Engineering

Security Architecture and Engineering focuses on designing, implementing, and maintaining secure systems, processes, and technologies.

### Key Areas

* Secure system design
* Security controls
* Firewalls
* Cryptography
* Infrastructure security

### Example

Configuring a firewall to monitor and filter incoming and outgoing network traffic.

---

## 4. Communication and Network Security

Communication and Network Security focuses on securing networks, communication channels, and data transmission.

### Key Areas

* Network security
* Wireless security
* Secure communication protocols
* Network monitoring
* Network architecture

### Example

Creating policies to prevent employees from connecting to unsecured public Wi-Fi networks.

---

## 5. Identity and Access Management (IAM)

Identity and Access Management focuses on ensuring that only authorized users can access organizational resources and data.

### Key Areas

* Authentication
* Authorization
* Access control
* Identity management
* Role-based access control (RBAC)

### Example

Managing employee keycard access and assigning appropriate system permissions.

---

## 6. Security Assessment and Testing

Security Assessment and Testing focuses on evaluating the effectiveness of security controls and identifying vulnerabilities.

### Key Areas

* Security audits
* Vulnerability assessments
* Penetration testing
* Security monitoring
* Compliance testing

### Example

Reviewing user permissions to ensure unauthorized employees cannot access payroll data.

---

## 7. Security Operations

Security Operations focuses on monitoring systems, responding to incidents, conducting investigations, and implementing preventive measures.

### Key Areas

* Incident response
* Threat detection
* Security monitoring
* Disaster recovery
* Investigations

### Example

Investigating alerts generated when an unknown device connects to the organization's network.

---

## 8. Software Development Security

Software Development Security focuses on integrating security practices throughout the Software Development Life Cycle (SDLC).

### Key Areas

* Secure coding practices
* Application security
* Code reviews
* Security testing
* Data protection

### Example

Ensuring a mobile application securely stores user data and enforces strong password requirements.

---

## Summary Table

| Domain                                | Primary Focus                              |
| ------------------------------------- | ------------------------------------------ |
| Security and Risk Management          | Risk, compliance, policies, and governance |
| Asset Security                        | Protection of physical and digital assets  |
| Security Architecture and Engineering | Secure system design and implementation    |
| Communication and Network Security    | Protection of networks and communications  |
| Identity and Access Management        | User authentication and authorization      |
| Security Assessment and Testing       | Auditing and evaluating security controls  |
| Security Operations                   | Monitoring and incident response           |
| Software Development Security         | Secure software development practices      |

## Key Takeaway

The eight CISSP security domains provide a comprehensive framework for managing organizational security. Security professionals often work across multiple domains because weaknesses in one area can impact the security of the entire organization.
