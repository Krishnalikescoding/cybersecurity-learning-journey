# SIEM Dashboards (Splunk & Google SecOps/Chronicle)

## What are SIEM Dashboards?

A **SIEM dashboard** is a visual interface that displays security data collected from logs.

Dashboards help security analysts:

- Monitor security events
- Detect threats
- Investigate incidents
- Analyze risks
- Track trends
- Improve security posture

---

# Splunk Dashboards

Splunk provides multiple dashboards to monitor an organization's security.

---

## 1. Security Posture Dashboard

### Purpose

Provides an overview of the organization's security over the last 24 hours.

### Displays

- Security events
- Threat trends
- Suspicious network activity
- Security alerts

### Used For

- Real-time monitoring
- Investigating suspicious IP addresses
- Verifying security controls

---

## 2. Executive Summary Dashboard

### Purpose

Provides a high-level overview of the organization's security health.

### Displays

- Incident trends
- Overall security status
- Risk summaries
- Security metrics

### Used For

- Reporting to executives
- Tracking long-term security performance
- Supporting business decisions

---

## 3. Incident Review Dashboard

### Purpose

Helps investigate security incidents.

### Displays

- Incident timelines
- High-risk alerts
- Event sequences
- Suspicious patterns

### Used For

- Incident investigation
- Threat analysis
- Prioritizing critical incidents

---

## 4. Risk Analysis Dashboard

### Purpose

Analyzes the risk associated with users, devices, or IP addresses.

### Risk Objects

- Users
- Computers
- IP addresses
- Devices

### Displays

- Unusual login activity
- Abnormal network traffic
- Changes in user behavior
- Risk scores

### Used For

- Prioritizing vulnerabilities
- Identifying high-risk assets
- Risk mitigation

---

# Google SecOps (Chronicle)

## What is Chronicle?

**Google SecOps (Chronicle)** is a **cloud-native SIEM** platform that collects, stores, searches, and analyzes security logs.

Chronicle can search by:

- Asset
- Domain name
- User
- IP address

---

# Chronicle Dashboards

---

## 1. Enterprise Insights Dashboard

### Purpose

Highlights recent security alerts and **Indicators of Compromise (IOCs)**.

### Displays

- Suspicious domain names
- Confidence scores
- Threat severity
- Recent alerts

### Used For

- Detecting malicious domains
- Monitoring critical assets
- Prioritizing threats

---

## 2. Data Ingestion and Health Dashboard

### Purpose

Monitors the health of log collection.

### Displays

- Number of event logs
- Log sources
- Processing success rate
- Data ingestion status

### Used For

- Verifying log collection
- Detecting ingestion failures
- Troubleshooting log sources

---

## 3. IOC Matches Dashboard

### Purpose

Tracks **Indicators of Compromise (IOCs)** across the organization.

### Displays

- Malicious domains
- Suspicious IP addresses
- Device IOCs
- Threat trends

### Used For

- Identifying active threats
- Monitoring attack patterns
- Prioritizing investigations

---

## 4. Main Dashboard

### Purpose

Provides a high-level overview of organizational security activity.

### Displays

- Data ingestion
- Alerts
- Security events
- Event timelines
- Threat trends

### Used For

- Daily security monitoring
- Identifying attack trends
- Monitoring overall security posture

---

## 5. Rule Detections Dashboard

### Purpose

Shows alerts generated by security detection rules.

### Displays

- Detection statistics
- Alert counts
- Severity levels
- Recurring incidents

### Used For

- Monitoring detection rules
- Identifying repeated attacks
- Improving mitigation strategies

---

## 6. User Sign-In Overview Dashboard

### Purpose

Monitors user authentication behavior.

### Displays

- User sign-ins
- Login locations
- Login timestamps
- Unusual login behavior

### Used For

- Detecting compromised accounts
- Identifying impossible travel
- Investigating suspicious logins

---

# Indicators of Compromise (IOCs)

### Definition

**Indicators of Compromise (IOCs)** are pieces of evidence that suggest a system or network may have been compromised.

### Examples

- Malicious IP addresses
- Suspicious domain names
- Malware file hashes
- Unusual login locations
- Unexpected network traffic

---

# Splunk vs Google SecOps (Chronicle)

| Feature | Splunk | Google SecOps (Chronicle) |
|---------|---------|---------------------------|
| Deployment | Self-hosted or Cloud | Cloud-native |
| Primary Use | Log analysis and monitoring | Large-scale cloud security analytics |
| Dashboards | Security posture, Incident review, Risk analysis, Executive summary | Enterprise insights, IOC matches, Data ingestion, Rule detections, User sign-in overview |
| Search Capability | Logs and events | Assets, users, domains, IP addresses |
| Best For | General SIEM operations | Cloud environments and threat intelligence |

---

# Key Takeaways

- **SIEM dashboards** present security data in an easy-to-understand visual format using charts, graphs, tables, and timelines.
- **Splunk** provides dashboards such as:
  - Security Posture
  - Executive Summary
  - Incident Review
  - Risk Analysis
- **Google SecOps (Chronicle)** provides dashboards such as:
  - Enterprise Insights
  - Data Ingestion and Health
  - IOC Matches
  - Main Dashboard
  - Rule Detections
  - User Sign-In Overview
- **Indicators of Compromise (IOCs)** help analysts identify malicious activity using evidence like suspicious IP addresses, domains, and file hashes.
- Security analysts use these dashboards to **monitor alerts, investigate incidents, analyze risks, prioritize threats, and strengthen an organization's security posture**.
