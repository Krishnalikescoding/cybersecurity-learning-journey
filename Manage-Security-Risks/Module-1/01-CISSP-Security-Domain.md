# CISSP Security Domains

As a cybersecurity analyst, you can explore various areas of cybersecurity that interest you. One way to understand these areas is through the **Certified Information Systems Security Professional (CISSP)** security domains. These eight domains organize the responsibilities of cybersecurity professionals and provide a framework for securing organizations.

---

# Domain 1: Security and Risk Management

This domain focuses on developing and maintaining an organization's **security posture**, which is its ability to protect critical assets and data while adapting to changing threats and risks.

## Elements of Security and Risk Management

- Security goals and objectives
- Risk mitigation processes
- Compliance requirements
- Business continuity plans
- Legal regulations
- Professional and organizational ethics

## Information Security (InfoSec)

Information Security (InfoSec) refers to the set of policies, procedures, and processes established to protect an organization's information from unauthorized access, modification, or destruction.

Organizations may create **playbooks**, conduct employee training, and implement security policies based on their business needs and risk levels.

### Common InfoSec Design Processes

- Incident response
- Vulnerability management
- Application security
- Cloud security
- Infrastructure security

### Example

A company operating in the European Union may need to change how it stores and processes **Personally Identifiable Information (PII)** to comply with the **General Data Protection Regulation (GDPR)**.

---

# Domain 2: Asset Security

Asset security focuses on protecting an organization's physical and digital assets throughout their lifecycle.

This includes managing:

- Data storage
- Maintenance
- Retention
- Backups
- Recovery
- Secure destruction of data

Organizations must keep track of all assets because losing or exposing them increases security risks.

Security analysts often perform tasks such as:

- Conducting security impact analyses
- Creating recovery plans
- Managing data exposure
- Performing backups
- Restoring systems after security incidents

Proper asset management helps ensure business continuity during cyber incidents.

---

# Domain 3: Security Architecture and Engineering

This domain focuses on designing secure systems, technologies, and processes that protect organizational assets and sensitive data.

Security architects and engineers are responsible for developing these secure designs.

## Shared Responsibility

Shared responsibility means everyone involved in designing, maintaining, and operating a system contributes to reducing security risks.

## Security Design Principles

- Threat modeling
- Principle of least privilege
- Defense in depth
- Fail securely
- Separation of duties
- Keep it simple
- Zero Trust
- Trust but verify

### Example

A Security Information and Event Management (**SIEM**) tool continuously monitors login attempts and user activities. If unusual login behavior is detected, it can alert analysts to a potential attack.

---

# Domain 4: Communication and Network Security

This domain focuses on protecting network infrastructure and communications.

It covers:

- Physical networks
- Wireless networks
- Cloud communications
- Remote access
- Hybrid work environments
- On-site communications

Organizations with remote employees must secure connections outside the office.

Security controls include:

- Restricted network access
- Firewalls
- Secure VPNs
- Network segmentation
- Secure wireless communication

These controls help protect users and organizational resources while employees work remotely.

---

# Domain 5: Identity and Access Management (IAM)

Identity and Access Management ensures that only authorized users can access organizational resources.

IAM focuses on:

- User identification
- Authentication
- Authorization
- Access control

Its goal is to prevent unauthorized access while allowing legitimate users to perform their responsibilities.

## Principle of Least Privilege

The **Principle of Least Privilege** means users receive only the minimum permissions required to complete their work.

This minimizes security risks if an account becomes compromised.

### Example

A customer service representative may be allowed to view a customer's phone number while resolving an issue but should lose access once the issue has been completed.

---

# Domain 6: Security Assessment and Testing

This domain focuses on identifying, evaluating, and reducing security risks, vulnerabilities, and threats.

Organizations regularly assess their security posture to determine whether systems are adequately protected.

## Common Activities

- Security assessments
- Vulnerability assessments
- Penetration testing (Pen Testing)
- Security control testing
- Security audits
- Collecting security data
- Analyzing security data
- Permission audits

Penetration testers simulate attacks to discover weaknesses before real attackers can exploit them.

Security analysts frequently audit user permissions to ensure employees have appropriate access levels.

---

# Domain 7: Security Operations

Security operations involve monitoring, detecting, responding to, and recovering from cybersecurity incidents.

Security teams work together to investigate attacks and strengthen organizational defenses.

## Security Operations Include

- Security awareness training
- Reporting
- Documentation
- Intrusion Detection Systems (IDS)
- Intrusion Prevention Systems (IPS)
- SIEM tools
- Log management
- Incident management
- Security playbooks
- Digital forensics
- Post-incident reviews
- Lessons learned

Security analysts monitor logs and alerts to detect suspicious activities, such as unauthorized access occurring outside business hours.

After an incident, they investigate what happened and recommend improvements to prevent future attacks.

---

# Domain 8: Software Development Security

Software Development Security focuses on creating secure software throughout the entire **Software Development Life Cycle (SDLC).**

Security must be considered during every phase:

- Planning
- Design
- Development
- Testing
- Deployment
- Maintenance

Security should never be treated as an afterthought.

## Activities

- Secure programming practices
- Code reviews
- Application security testing
- Vulnerability scanning
- Penetration testing
- Encryption implementation
- Quality assurance testing

### Example

A cybersecurity analyst working for a pharmaceutical company may verify that encryption is correctly configured for a medical device that stores sensitive patient information.

# Important Concepts

## Information Security (InfoSec)

Information Security (InfoSec) is the collection of policies, procedures, technologies, and processes used to protect organizational information from unauthorized access, disclosure, modification, or destruction.

Examples include:

- Incident response
- Vulnerability management
- Cloud security
- Infrastructure security
- Application security

---

## Principle of Least Privilege

The Principle of Least Privilege is the practice of giving users only the minimum permissions necessary to complete their work.

Benefits include:

- Reduces attack surface
- Limits damage if an account is compromised
- Prevents unauthorized access
- Improves overall security

---

# Key Takeaways

- CISSP organizes cybersecurity into **eight security domains**.
- Security and Risk Management develops an organization's overall security posture.
- Asset Security protects data and physical assets throughout their lifecycle.
- Security Architecture and Engineering focuses on designing secure systems.
- Communication and Network Security protects organizational networks.
- Identity and Access Management ensures only authorized users can access resources.
- Security Assessment and Testing identifies vulnerabilities before attackers exploit them.
- Security Operations detects, investigates, and responds to security incidents.
- Software Development Security ensures security is integrated into every stage of software development.
- Information Security (InfoSec) protects organizational information through policies, procedures, and technologies.
- The Principle of Least Privilege limits user permissions to only what is necessary, reducing overall security risks.
