# Course 2 - Module 1: Security Frameworks, Controls, and Risk

## Terms and Definitions

| Term | Definition |
|------|------------|
| **Assess** | The fifth step of the NIST RMF that determines if established controls are implemented correctly. |
| **Authorize** | The sixth step of the NIST RMF where leadership accepts accountability for security and privacy risks. |
| **Business Continuity** | An organization's ability to continue operations through disaster recovery and risk management planning. |
| **Categorize** | The second step of the NIST RMF used to classify systems and develop risk management processes. |
| **External Threat** | Any threat outside the organization that can harm organizational assets. |
| **Implement** | The fourth step of the NIST RMF where security and privacy controls are put into operation. |
| **Internal Threat** | A current/former employee, vendor, or trusted partner who poses a security risk. |
| **Monitor** | The seventh step of the NIST RMF involving continuous monitoring of systems and controls. |
| **Prepare** | The first step of the NIST RMF involving planning activities before security incidents occur. |
| **Ransomware** | Malware that encrypts data and demands payment to restore access. |
| **Risk** | Anything that can affect the confidentiality, integrity, or availability (CIA) of an asset. |
| **Risk Mitigation** | The process of reducing the impact of security risks through policies and controls. |
| **Security Posture** | An organization's ability to defend assets, manage risks, and respond to threats. |
| **Select** | The third step of the NIST RMF where security controls are chosen and documented. |
| **Shared Responsibility** | The principle that everyone in an organization helps maintain security. |
| **Social Engineering** | Manipulating people into revealing sensitive information or granting unauthorized access. |
| **Vulnerability** | A weakness that can be exploited by a threat actor. |

# Security Frameworks

Security frameworks are structured guidelines used to:

- Mitigate risks and threats
- Protect data and privacy
- Meet compliance requirements
- Build organizational security policies
- Improve overall security posture

Frameworks help organizations:

- Prevent cyber attacks
- Detect security incidents
- Respond to breaches
- Recover from attacks

### Examples of threats addressed

- Social engineering
- Phishing
- Ransomware
- Unauthorized access
- Physical security breaches

Employee awareness and security training are essential because **people are often the weakest link in security**.

---

# Security Controls

Security controls are safeguards that reduce specific security risks.

Controls work alongside security frameworks to:

- Prevent attacks
- Detect threats
- Correct security issues
- Protect organizational assets

Without proper controls, organizations may experience:

- Financial loss
- Reputation damage
- Data breaches
- Unauthorized access
- Identity fraud

---

# Common Security Controls

## 1. Encryption

Encryption converts readable **plaintext** into unreadable **ciphertext**.

Purpose:

- Protect confidential information
- Prevent unauthorized reading of data

Examples:

- Customer account information
- Social Security numbers
- Financial records

---

## 2. Authentication

Authentication verifies the identity of a user or device.

Examples:

- Username and password
- Multi-Factor Authentication (MFA)
- Biometrics

### Multi-Factor Authentication (MFA)

Requires two or more verification methods, such as:

- Password
- Security code
- Fingerprint
- Face scan
- Voice recognition

### Biometrics

Unique physical characteristics used for identity verification.

Examples:

- Fingerprint
- Iris scan
- Palm scan
- Face recognition

### Vishing

Voice phishing that impersonates trusted individuals using phone calls or voice technology to steal sensitive information.

---

## 3. Authorization

Authorization determines **what resources a verified user is allowed to access**.

Example:

- A government security analyst accessing classified internal systems after authentication.

---

# CIA Triad

Security controls support the CIA Triad:

- **Confidentiality** – Protect information from unauthorized access.
- **Integrity** – Ensure data remains accurate and unaltered.
- **Availability** – Keep systems and data accessible when needed.

---

# Frameworks and Controls

Frameworks provide guidance for managing cybersecurity.

Controls implement the protections required by those frameworks.

Together they help organizations:

- Reduce risks
- Protect assets
- Meet legal requirements
- Improve security posture

---

# Cyber Threat Framework (CTF)

Developed by the U.S. government.

Purpose:

- Provide a common language for cyber threats.
- Improve communication among cybersecurity professionals.
- Support faster threat analysis and response.

Benefits:

- Better threat intelligence sharing
- Improved incident response
- Standardized reporting

---

# ISO/IEC 27001

An internationally recognized information security framework.

Purpose:

- Manage information security risks.
- Protect organizational assets.
- Establish an Information Security Management System (ISMS).

Protects:

- Financial information
- Intellectual property
- Employee information
- Third-party data

ISO/IEC 27001 provides recommended security controls but does not require specific ones.

---

# Key Takeaways

- Security frameworks provide guidance for managing cybersecurity.
- Security controls reduce specific risks.
- Frameworks and controls work together to improve security posture.
- Organizations use them to comply with regulations and protect critical assets.
- Employee awareness and training are essential for defending against social engineering attacks.
- The NIST RMF consists of seven steps:
  1. Prepare
  2. Categorize
  3. Select
  4. Implement
  5. Assess
  6. Authorize
  7. Monitor
- Common security controls include encryption, authentication, authorization, and MFA.
- Controls can be physical, technical, or administrative.
- The CIA Triad remains the foundation of information security.
