# Module 2: NIST Cybersecurity Framework (CSF) Core Functions

> **Note:** This lesson describes the original five CSF functions. In **NIST CSF v2.0**, a sixth function, **Govern**, was added and comes before **Identify**.

---

# NIST CSF Core Functions

The **NIST Cybersecurity Framework (CSF)** helps organizations:

- Manage cybersecurity risks
- Protect critical assets
- Detect security incidents
- Respond effectively
- Recover quickly
- Improve security over time

---

# 1. Identify

**Purpose:**
Understand the organization's assets, systems, risks, and vulnerabilities.

### Activities

- Inventory hardware and software
- Identify critical assets
- Assess cybersecurity risks
- Monitor systems and devices
- Understand business environment

### Analyst Responsibilities

- Monitor internal networks
- Identify potential security issues
- Document organizational assets

---

# 2. Protect

**Purpose:**
Implement safeguards to reduce cybersecurity risks.

### Activities

- Access control
- Security awareness training
- Encryption
- Multi-Factor Authentication (MFA)
- Security policies and procedures

### Analyst Responsibilities

- Improve security policies
- Study previous incidents
- Recommend stronger protections
- Implement security tools

---

# 3. Detect

**Purpose:**
Quickly identify cybersecurity events and security incidents.

### Activities

- Continuous monitoring
- Log analysis
- SIEM monitoring
- Intrusion detection
- Alert generation

### Analyst Responsibilities

- Configure security monitoring tools
- Review alerts
- Classify incidents (Low, Medium, High)
- Notify the security team

---

# 4. Respond

**Purpose:**
Contain, investigate, and minimize the impact of security incidents.

### Activities

- Incident response
- Threat containment
- Root cause analysis
- Evidence collection
- Process improvement

### Analyst Responsibilities

- Document incidents
- Collect security logs and evidence
- Work with the incident response team
- Recommend improvements to prevent future attacks

---

# 5. Recover

**Purpose:**
Restore systems and return normal business operations after a security incident.

### Activities

- Restore backups
- Recover systems and data
- Repair affected devices
- Resume business operations
- Improve recovery plans

### Analyst Responsibilities

- Help restore affected systems
- Recover important files
- Verify systems are operating normally
- Support business continuity

---

# CSF Workflow

```text
Identify
    ↓
Protect
    ↓
Detect
    ↓
Respond
    ↓
Recover
```

**CSF v2.0 Workflow**

```text
Govern
    ↓
Identify
    ↓
Protect
    ↓
Detect
    ↓
Respond
    ↓
Recover
```

---

# Why the CSF Is Important

The NIST CSF helps organizations:

- Reduce cybersecurity risks
- Protect critical assets
- Improve incident response
- Strengthen security posture
- Support business continuity
- Continuously improve cybersecurity practices

---

# Key Takeaways

- The NIST CSF provides a structured approach to cybersecurity risk management.
- The original framework has **five core functions**:
  1. Identify
  2. Protect
  3. Detect
  4. Respond
  5. Recover
- **CSF v2.0** adds a sixth function, **Govern**, to strengthen cybersecurity governance.
- The framework combines **proactive** measures (Identify, Protect) with **reactive** measures (Detect, Respond, Recover).
- Security analysts play a role in every stage, from identifying risks to restoring systems after incidents.
- Continuous improvement after every incident helps organizations become more resilient against future threats.
