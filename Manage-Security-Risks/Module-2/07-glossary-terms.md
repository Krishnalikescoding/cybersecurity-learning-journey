# Terms and Definitions

| **Term** | **Definition** |
|-----------|----------------|
| **Asset** | An item perceived as having value to an organization. |
| **Attack Vectors** | The pathways attackers use to penetrate security defenses. |
| **Authentication** | The process of verifying who someone is. |
| **Authorization** | The concept of granting access to specific resources in a system. |
| **Availability** | The idea that data is accessible to authorized users when needed. |
| **Biometrics** | Unique physical characteristics used to verify a person's identity. |
| **Confidentiality** | The idea that only authorized users can access specific assets or data. |
| **Confidentiality, Integrity, Availability (CIA) Triad** | A security model that helps organizations manage risk and protect information through confidentiality, integrity, and availability. |
| **Detect** | A NIST CSF core function focused on identifying security incidents and improving monitoring capabilities. |
| **Encryption** | The process of converting readable data (plaintext) into encoded data (ciphertext). |
| **Govern** | A NIST CSF v2.0 core function that establishes and oversees cybersecurity strategy, policies, roles, and risk management aligned with business goals and regulations. |
| **Identify** | A NIST CSF core function that manages cybersecurity risks affecting organizational people, assets, and systems. |
| **Integrity** | The assurance that data is accurate, authentic, and reliable. |
| **National Institute of Standards and Technology (NIST) Cybersecurity Framework (CSF)** | A voluntary framework consisting of standards, guidelines, and best practices for managing cybersecurity risk. |
| **National Institute of Standards and Technology (NIST) Special Publication (SP) 800-53** | A framework providing security and privacy controls for U.S. federal information systems. |
| **Open Web Application Security Project (OWASP)** | A non-profit organization focused on improving software and web application security through best practices and resources. |
| **Protect** | A NIST CSF core function that implements safeguards, policies, procedures, training, and technologies to reduce cybersecurity risks. |
| **Recover** | A NIST CSF core function focused on restoring systems and normal operations after a cybersecurity incident. |
| **Respond** | A NIST CSF core function that contains, analyzes, and mitigates security incidents while improving response processes. |
| **Risk** | Anything that can affect the confidentiality, integrity, or availability (CIA) of an asset. |
| **Security Audit** | A review of an organization's security controls, policies, and procedures against established standards and expectations. |
| **Security Controls** | Safeguards designed to reduce specific security risks. |
| **Security Frameworks** | Guidelines used to build plans that mitigate risks and threats to data and privacy. |
| **Security Posture** | An organization's ability to protect critical assets, manage risks, and respond to changing threats. |
| **Threat** | Any circumstance or event that can negatively impact organizational assets. |
